# Post para LinkedIn — @igoruehara/spec-driven

🚀 Abri o código de uma CLI que estrutura **Spec-Driven Development** pro Claude Code.

Um comando — `npx @igoruehara/spec-driven` — e seu projeto nasce com uma esteira completa:
da **descoberta do produto** até o **código rastreável**.

O problema: IA codando sem contexto vira retrabalho. Spec solta vira caos.
A ideia: **a spec é o contrato.** Tudo deriva dela — e o agente segue.

O que vem dentro 👇
🧭 **Lean Inception** — visão, personas, jornadas, funcionalidades sequenciadas, MVP
🧩 **DDD** — bounded contexts, linguagem ubíqua, diagramas (Mermaid)
📐 **Technical Design Docs** — 5 eixos: stack, arquitetura, infra, qualidade, observabilidade
✅ **SDD** — critérios de aceite testáveis viram o contrato *e* o prompt do agente
🤖 **14 skills** (/kickoff, /nova-feature, /validar, /metricas…) + MCPs (Jira, Confluence) com trava de conta/workspace

O diferencial: **a esteira se valida sozinha.**
🔎 Auditoria estrutural + eval de fidelidade spec→código rodando na **CI**
📊 Lead Time, Throughput e maturidade de Continuous Delivery/Deployment
🧪 Exemplo completo provado ponta a ponta (audit + eval verdes, testes passando)

100% open source. Use, forke, melhore:
👉 github.com/igoruehara/spec-driven

Como você estrutura contexto pra IA codar com qualidade? Comenta aí. 💬
